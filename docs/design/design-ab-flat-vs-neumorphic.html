Flat vs Tasy Neumorphic — os dois contratos de design do Vitali

O repo tem dois documentos de design que se contradizem sobre a estética. Este mock renderiza os mesmos elementos nas duas linguagens, lado a lado, com a recipe exata de cada uma. Coluna esquerda = o que o DESIGN.md manda. Coluna direita = o que está em produção.

A · Flat "Clinical-clean SaaS"

DESIGN.md v2.0.0 · commit f1c9355 (18/jun) · 278 linhas

Tasy citado só como benchmark de UX (§ "Enterprise/Tasy-grade UX contract"): denso, integrado, status-first. Estética própria: flat, blue-600, slate-*, rounded-lg, sem sombra.

Aplicado em: components/shared/*, lib/operational-ui.ts, telas reconciliadas (agenda, sala de espera, workbench TISS).

B · Tasy Neumorphic

docs/FRONTEND_GUIDELINES.md · commit f1c9355 (18/jun) · 58 componentes

Identidade visual explícita: "ERP Médico Corporativo + Neumorfismo Suave". #0066A1 em gradiente, fundos #DFE5EB/#EBF0F5, sombras inset, rounded-xl, font-bold, text-xs.

Golden standard: frontend/app/setup/page.tsx. Vazou pros primitivos flat via #F4F7FA / #24292F / #57606A / #8C959F.

7. Onde exatamente eles se contradizem

Dimensão A · Flat (DESIGN.md) B · Neumorphic (GUIDELINES) Conflito
Elevação Flat. shadow-sm só em modal / painel sticky Sombra inset + outer em tudo Direto
Raio rounded-lg — "nunca rounded-xl" rounded-xl em painéis, rounded-2xl no shell Direto
Primária blue-600 chapado (#2563EB) Gradiente #0066A1 → #005282 Direto
Peso de fonte font-semibold — font-bold é Don't font-bold em quase tudo Direto
Título de página <h1> text-2xl font-semibold "não use maior que text-base" Direto
Input Borda slate-200, fundo branco, py-2 Escavado #E8EDF2, sem borda, h-8 Direto
Fundo de app slate-50 / #F4F7FA #DFE5EB → #EBF0F5 → #F4F7FA Parcial
Mapas de status lib/operational-ui como fonte única Não redefine status Compatível
Densidade / UX Denso, status-first, tabela > card Denso, grid-cols-12, text-xs Compatível
Ícones lucide-react, nav 18 / inline 14-16 lucide-react, 14 ou 16 Compatível

Leitura

O conflito é só de chrome — elevação, raio, peso, gradiente, tratamento de input. A camada de significado (mapas de status, labels PT canônicos, densidade, status-first, tabela sobre card) é idêntica nos dois. Isso quer dizer que reconciliar não exige rediscutir UX: é escolher uma pele e aplicar.
