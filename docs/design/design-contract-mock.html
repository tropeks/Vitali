Divergência doc ↔ código

O DESIGN.md descreve tudo em slate-*, mas os primitivos usam hex arbitrários: #F4F7FA (bg de página, tile neutro), #8C959F (label de KPI neutro), #57606A (texto de SectionState neutro), #24292F (título de ReadinessPanel). Este mock reproduz o código. Vale decidir qual vence e alinhar os dois.

Nunca decorativa. Todo status resolve para um destes tons via lib/operational-ui.

8. Do / Don't

Do

  • • <PageShell variant> em toda tela
  • • Resolver status pelos mapas de lib/operational-ui
  • • StatusBadge / KpiTile / SectionState / ReadinessPanel
  • • rounded-lg border border-slate-200, flat
  • • <h1> text-2xl font-semibold como título de página
  • • Label canônico em português
  • • font-mono em todo código médico

Don't

  • • Escrever min-h-full / max-w / space-y na mão
  • • Ternário status→cor inline no call site
  • • Reimplementar pill / tile / empty / prontidão por página
  • • rounded-xl, shadow-sm em card estático, borda gray-200
  • • <h2> como título de página, font-bold em título ou KPI
  • • Deixar status_display do servidor mandar num status conhecido
  • • Cor semântica sem texto ao lado · animação de entrada · serifada
